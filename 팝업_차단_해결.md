# 카카오 로그인 팝업 차단 해결 가이드

`Auth.login() 호출 완료` 후 콜백이 호출되지 않는 경우, **팝업이 차단**되었을 가능성이 높습니다.

---

## 증상

- ✅ SDK 초기화 성공
- ✅ JS Key 설정됨
- ✅ `Auth.login()` 호출 완료
- ❌ **콜백이 호출되지 않음** (10초 타임아웃)

→ **팝업이 차단되었거나 열리지 않음**

---

## 해결 방법

### 1. 브라우저 팝업 차단 해제 (가장 중요!)

#### Chrome / Edge
1. 카카오 로그인 버튼 클릭
2. 브라우저 **주소창 오른쪽**에서 **팝업 차단 아이콘** 확인
3. 아이콘 클릭 → **"항상 localhost의 팝업 및 리다이렉트 허용"** 선택
4. 다시 카카오 로그인 버튼 클릭

#### Firefox
1. 주소창 왼쪽의 **자물쇠 아이콘** 클릭
2. **권한** → **팝업 차단** → **허용** 선택

#### Safari
1. Safari → 환경설정 → 웹사이트 → 팝업 윈도우
2. localhost → **허용** 선택

---

### 2. 브라우저 설정에서 팝업 허용

#### Chrome
1. 설정 (chrome://settings/)
2. **개인정보 및 보안** → **사이트 설정**
3. **팝업 및 리다이렉트**
4. **허용** 목록에 `http://localhost:3000` 추가

---

### 3. 시크릿 모드에서 시도

브라우저 확장 프로그램이 팝업을 차단할 수 있습니다:
- **Ctrl + Shift + N** (Chrome/Edge)
- **Ctrl + Shift + P** (Firefox)
- 시크릿 모드에서 `http://localhost:3000/login` 접속 후 카카오 로그인 시도

---

### 4. 다른 브라우저에서 시도

브라우저별로 동작이 다를 수 있습니다:
- Chrome
- Edge
- Firefox

---

### 5. 카카오 개발자 콘솔 설정 확인

팝업이 열리더라도 카카오 앱 설정이 잘못되면 콜백이 호출되지 않을 수 있습니다:

1. [카카오 개발자 콘솔](https://developers.kakao.com/) 접속
2. **내 애플리케이션** → 앱 선택
3. **앱 설정** → **플랫폼** 탭
4. **웹 플랫폼** 등록 확인
5. **사이트 도메인**에 다음 추가:
   - `http://localhost:3000`
   - `http://127.0.0.1:3000`
6. **카카오 로그인** 탭 → **Redirect URI**에 다음 추가:
   - `http://localhost:3000`
   - `http://127.0.0.1:3000`

---

## 팝업이 열리는지 확인하는 방법

1. 카카오 로그인 버튼 클릭
2. **새 창/탭이 열리는지** 확인
3. 또는 **작은 팝업 창**이 열리는지 확인
4. 팝업이 열리면 → 카카오 로그인 완료
5. 팝업이 안 열리면 → **팝업 차단 해제** 필요

## 팝업이 여전히 안 열리면

카카오 SDK v1.43.6은 구버전입니다. 최신 SDK로 업데이트하거나, 리다이렉트 방식으로 변경하는 것을 고려해볼 수 있습니다.

현재는 **팝업 차단 해제**가 가장 확실한 해결 방법입니다.
